### Assumptions documented

- Couple names at weight 300 (lightest Cormorant Garamond weight loaded) for maximum elegance at large display size. Executor may try weight 400 if 300 renders too thin on low-DPI screens.
- "Save the Date" label treated as an uppercase Jost label (tracking 0.25em), consistent with the muted/uppercase-tracking-text role in CLAUDE.md.
- Date and location use distinct fonts (display vs. body) to create visual hierarchy: date is the key fact (display serif), location is supporting context (body sans).

## Color

Source: CLAUDE.md (binding), src/index.css (tokens already defined as CSS custom properties).

| Role | Token | Hex | Usage in this phase |
|------|-------|-----|---------------------|
| Dominant (60%) | `--forest` | #0B1610 | Full-bleed background behind hero; visible where hero does not cover (letterbox safety) |
| Secondary (30%) | `--forest` (same) | #0B1610 | No distinct secondary surface in Phase 2 — single-layer full-bleed composition |
| Accent — display (15%) | `--gold-light` | #D4B57A | Couple names "Aaron & Rina" only |
| Accent — labels/borders (10%) | `--gold` | #BF9B5A | "SAVE THE DATE" label; divider line |
| Body text (20%) | `--cream` | #EAE0CB | Date, location |
| Subdued / footer | `--muted` | #72685A | "Formal invitation to follow" footer note |
| Hero scrim | n/a — CSS gradient | linear-gradient(to top, rgba(11,22,16,0.82) 0%, rgba(11,22,16,0.45) 40%, rgba(11,22,16,0.20) 70%, transparent 100%) | Overlay on hero image to ensure text legibility |

### Color assumptions documented

- **Scrim values:** The hero photo is a real photograph over a dark forest scene. The radial/gradient scrim uses `--forest` color (#0B1610) at 82% opacity at the bottom (where the text block sits) fading to transparent at 70% height. This ensures WCAG AA contrast for cream text on the gradient zone without fully obscuring the photo.
- **No distinct secondary surface:** Phase 2 is a single full-bleed composition. There is no card, sidebar, or nav surface — the 60/30/10 split collapses to dominant + accents.

Accent reserved for:
- `--gold-light` (#D4B57A): Couple names exclusively
- `--gold` (#BF9B5A): "SAVE THE DATE" label text and the horizontal divider line exclusively

## Layout & Composition

### Page structure

The page is a single full-viewport composition:

```
[  viewport: 100vw × 100svh  ]
├── .heroContainer     (position: absolute; inset: 0; z-index: 0)
│   └── img            (width: 100%; height: 100%; object-fit: cover; object-position: center)
├── .scrim             (position: absolute; inset: 0; z-index: 1; gradient overlay)
└── .contentBlock      (position: absolute; bottom: 0; left: 0; right: 0; z-index: 2; padding: 0 xl mb 2xl)
    ├── .label          "SAVE THE DATE"
    ├── .coupleNames    "Aaron & Rina"
    ├── .divider        <hr> or <div> — 1px horizontal line, 48px wide, centered
    ├── .date           "May 30, 2027"
    ├── .location       "Oahu, Hawaii"
    └── .footer         "Formal invitation to follow"
```

### Alignment

- All content: horizontally centered (`text-align: center`)
- Content block anchored to viewport bottom with bottom padding of 48px (desktop) / 32px (mobile)
- `.app` outer container: `position: relative; min-height: 100svh; overflow: hidden; background-color: var(--forest)`

### Hero image

- Source: `/images/save-the-date-hero.png`
- Technique: `<img>` tag with `width: 100%; height: 100%; object-fit: cover; object-position: center`
- Container: `position: absolute; inset: 0; overflow: hidden`
- No Ken Burns / parallax in Phase 2 — static image only (those are Phase 4 concerns)
- Background fallback: `background-color: var(--forest)` on `.app` if image fails to load

### Divider line

- Rendered as a `<div>` with `width: 48px; height: 1px; background-color: var(--gold); margin: 24px auto`
- Not `<hr>` (avoid browser UA stylesheet normalization conflicts in CSS Modules)

### Vertical spacing rhythm (content block, top to bottom)

| Gap | Tokens | Value |
|-----|--------|-------|
| Label bottom margin | sm | 8px |
| Couple names bottom margin | lg | 24px |
| Divider total height (line + margins) | lg + lg | 48px (24px margin-top auto + 1px + 24px margin-bottom) |
| Date bottom margin | md | 16px |
| Location bottom margin | xl | 32px |
| Footer (bottom of content block) | — | flush |

## Hero Image Overlay Contract

The scrim layer sits between the hero `<img>` and the content block. It must be implemented exactly:

```css
.scrim {
  position: absolute;
  inset: 0;
  z-index: 1;
  background: linear-gradient(
    to top,
    rgba(11, 22, 16, 0.82) 0%,
    rgba(11, 22, 16, 0.45) 40%,
    rgba(11, 22, 16, 0.20) 70%,
    transparent 100%
  );
  pointer-events: none;
}
```

Rationale: The content block sits in the bottom 30% of the viewport. The 82% forest-color opacity at the base provides at minimum 4.5:1 contrast ratio for cream text (#EAE0CB) against the blended surface, satisfying WCAG AA. The gradient fades to transparent at the top so the photo is visible and unobscured in the upper 30%.

## Assumptions & Open Questions

All assumptions were made because the scope excludes interactivity and CLAUDE.md fully defines the design system. No user questions were needed.

| Assumption | Rationale | Override if |
|-----------|-----------|-------------|
| Couple names font-weight: 300 | Lightest loaded weight; maximally elegant at 72px display | Renders too thin on 1x screens → try weight 400 |
