Flags (parse from process.argv):
- --resend (alias --force): re-send even if invited_at is set (refresh timestamp).
- --dry-run: print exactly who WOULD be emailed; send nothing; write nothing to DB.

Fail-fast guards (clear messages, exit 1), BEFORE any DB/SMTP work:
- DATABASE_URL missing -> message.
- GMAIL_USER missing -> message.
- GMAIL_APP_PASSWORD missing -> message.
- links.csv unreadable -> "links.csv not found — run: npm run db:generate-links first"
  (mirror check-link.js error). Read 'links.csv' (no path arg needed).
NOTE: do NOT read or require GUEST_TOKEN_SECRET — this script never touches it.

Parse links.csv with `parse(raw, { columns: true })` (same as check-link.js).
Validate expected columns id, display_name, email, url (mirror check-link.js
EXPECTED_COLUMNS check) and exit 1 with a clear message if wrong.

Build the transport (skip entirely in --dry-run):
```js
const transport = nodemailer.createTransport({
  service: 'gmail',
  auth: { user: process.env.GMAIL_USER, pass: process.env.GMAIL_APP_PASSWORD },
});
```
From: "Rina & Aaron <rinaaron5@gmail.com>".

Per-row loop (idempotency is the core requirement — per EMAIL-01):
1. Look up DB state by id: `SELECT invited_at, deleted_at FROM guests WHERE id = ${row.id}`.
   If no DB row found, treat as skip with a warning (id not in DB).
2. If deleted_at IS NOT NULL -> ALWAYS skip (count: skippedDeleted). Continue.
3. If invited_at IS NOT NULL AND NOT --resend -> skip (count: skippedInvited). Continue.
4. Otherwise this row is a send candidate.
   - If --dry-run: print "WOULD email {display_name} <{email}> -> {url}", count as
     wouldSend, write NOTHING to DB, send NOTHING. Continue.
   - Else: const { subject, html, text } = renderInvite({ displayName: row.display_name, url: row.url });
     Wrap the send in try/catch:
       try {
         await transport.sendMail({ from, to: row.email, subject, html, text });
         await sql`UPDATE guests SET invited_at = now() WHERE id = ${row.id}`;  // ONLY after success
         sent++;
       } catch (err) {
         console.error(`FAILED to send to ${row.email}: ${err.message}`);
         failed++;  // do NOT set invited_at — retried next run
       }
   CRITICAL: set invited_at = now() ONLY AFTER a successful send, so a crash
   mid-run never marks an unsent guest as sent.

Friendly summary at the end (mirror generate-links.js tone):
- Normal: `Done! ${sent} sent, ${skippedInvited} skipped (already invited), ${skippedDeleted} skipped (deleted).` and if failed>0 add `${failed} failed (will retry next run).`
- Dry-run: `Dry run: ${wouldSend} would be emailed, ${skippedInvited} already invited, ${skippedDeleted} deleted. Nothing sent, nothing written.`

OUT OF SCOPE: no bulk/queue, no retry backoff, no unsubscribe, no scheduling.
